## 任务

WAF 报告资源识别 v2

## 背景

当前 WAF Word 审计链对 CPU / 内存 / 磁盘资源项的识别仍偏粗糙：
- `无异常值` 会被“异常”二字误伤
- 仅有百分比、缺少“正常/异常”词时无法稳定落 claim
- 还没有贴合现有 WAF 报告的三列表格结构

用户提供的真实报告中，典型资源行形态为：
- `节点负载状态 | CPU | 使用率稳定在4.38%左右，无异常值`
- `节点负载状态 | 内存 | 使用率稳定在72.99%左右，无异常值`
- `磁盘状态 | 磁盘使用率 | 硬盘空间大小：50.91 TB 使用率：77.71%`

## 本轮目标

1. 让 WAF Word 报告中的 CPU / 内存 / 磁盘使用率能被稳定识别
2. 先正确识别文档资源数据，再进入日志核验
3. 保持现有审计主链不变，只增强资源 claim 提取

## 实施点

1. 在 `report_claim_normalizer.py` 中补资源行级识别
   - 贴合三列表格资源行
   - 优先从具体列识别 metric 与 detail
2. 修复 `无异常值 / 未见异常 / 稳定 / 空间充足` 这类文案误判
3. 在缺少显式“正常/异常”时，允许按百分比阈值推断资源级别
   - CPU
   - 内存
   - 磁盘
4. 增加与真实 WAF 报告结构接近的单测
5. 本地验证真实报告的 claim 提取结果

## 验收

- CPU 4.38% + 无异常值 -> `cpu / normal`
- 内存 72.99% + 无异常值 -> `memory / normal`
- 磁盘 77.71% -> `disk / normal`
- 高百分比但无显式异常词 -> 仍可识别为异常资源 claim
